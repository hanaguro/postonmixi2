# PostOnMixi2 開発学習Wiki

このWikiは、ソースコード内の実装意図と、そこで使用されているGoおよびLua/Neovim APIの知識をまとめたものです。

## 🚀 Go言語 実装メモ

### 変数宣言と制御
- **短縮変数宣言 (`:=`)**: 型指定なしで変数を宣言し、同時に値を代入します。関数内でのみ使用可能です。
- **defer 文**: `defer f.Close()` のように記述すると、その関数が終了する直前（return時など）に指定した処理が実行されます。リソースの解放漏れを防ぐために必須のパターンです。

### ファイル・環境操作
- **パスの展開**: `os.UserHomeDir()` を使用して、設定ファイルパスに含まれる `~/`（ホームディレクトリ）を実際の絶対パスに変換しています。
- **エラー返却**: Goの慣習に従い、関数の最後に `return scanner.Err()` のように、正常終了時は `nil`、異常時は `error` を返します。

## 🌙 Lua / Neovim API 実装メモ

### 外部プロセス実行 (`vim.fn.jobstart`)
- **バッファリング (`stdout_buffered = true`)**: 標準出力をリアルタイムではなく、プロセス終了時にまとめて一度に取得する設定です。
- **コールバック引数**: `function(_, data)` のように `_` を使うことで、使用しない引数（job_id等）を明示的に無視しています。
- **文字列結合**: `table.concat(data, "")` を使い、行ごとに分割されて届いた出力データを一つの文字列にまとめてから `vim.trim()` で空白を除去しています。

### ファイルシステムとパス操作
- **パス展開 (`vim.fn.expand`)**: `~/.config/...` のようなチルダを含むパスを絶対パスに変換します。
- **ディレクトリ名取得 (`fnamemodify`)**: 修飾子 `:h` (head) を指定することで、ファイルパスから親ディレクトリ部分のみを抽出します。
- **ディレクトリ作成 (`vim.fn.mkdir`)**: 第2引数に `"p"` を指定することで、親ディレクトリも含めて再帰的に作成します。

### テキスト処理とバッファ管理
- **最短一致パターン (`(.-)`)**: Luaのパターンマッチングにおいて、条件に合う最小限の範囲を抽出します。
- **文字数カウント (`vim.fn.strchars`)**: 単なるバイト数ではなく、Unicode文字数を正しくカウントし、mixi2の文字数制限(149字)を判定しています。
- **変更フラグ (`modified`)**: `vim.api.nvim_buf_set_option(buf, "modified", false)` とすることで、保存されていない変更があるという警告を出さずにバッファを閉じることができます。

### 非同期処理とUI
- **メインループへの予約 (`vim.schedule`)**: 非同期処理（jobstartのコールバック等）の中でUI操作（ウィンドウを閉じる等）を行う場合、`vim.schedule` を使ってNeovimのメインイベントループで実行させる必要があります。これをしないとAPIエラーになることがあります。
- **強制終了 (`nvim_win_close`)**: 第2引数に `true` を渡すことで、保存確認をスキップしてウィンドウを強制的に閉じます。

### キーマップとユーザーコマンド
- **再帰防止 (`noremap = true`)**: 設定したキーが別の設定を呼び出して無限ループになるのを防ぎます。
- **サイレント実行 (`silent = true`)**: 実行時にコマンドラインに「何を実行したか」というメッセージを表示させません。
- **ユーザーコマンドの引数 (`nargs = "+"`)**: `:PostOnMixi2` コマンドにおいて、最低1つ以上の引数（`auth` や `send` など）を必須にする設定です。
- **長さ演算子 (`#p`)**: Luaにおいてテーブル（配列）の要素数を取得します。
